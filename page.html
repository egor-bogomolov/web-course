Простые селекторы

Позволяют выбрать множество тегов в html-файле и задать для них некоторые параметры, например, выделить курсивом , изменить цвет и много всего другого.

  • Выбор тега: span { color: red; }
  • Задание класса или нескольких: .blue { color: blue; }
  • Задание идентификатора: #green { color: green; }
  • Можно перечислить через запятую несколько групп: #green, #also_green { color: green; }

Идентификатор должен встречаться на странице только один раз, иначе поведение не гарантируется.

Один тег можно отнести сразу к нескольким классам. Они перечисляются через пробел: class="code blue"

Сложные селекторы

  • Тег следует за заданным: h2 + p { color: darkmagenta; }
  • Вложенность, может быть не только в тег: ul.parent span { color: olive; }
  • Непосредственный ребенок: li.parent > span { color: darkgoldenrod; }
  • Комбинация вышеописанного: li.parent + li span { color: darkcyan; }
  • Еще много всего, можно посмотреть по ссылке ниже.

Подобные конструкции стоит использовать, с их помощью часто можно легко описать тот набор тегов, который требуется изменить. Это предпочтительнее, чем использование исключительно классов и индентификаторов, поскольку на больших страницах позволяет избежать ошибок из серии "забыл добавить класс у одного тега".

Если для тега не определены некоторые свойства, например, цвет текста, то он наследует их у тега, в который он вложен. Если ни у одного тега, лежащего выше по дереву, они также не определены, то используются стандартные свойства для браузера. В случае с отступами наследование не происходит, а сразу используются стандартные значения для браузера, если не сказано иного.

Для одного тега может быть несколько правил в CSS файле, которые задают ему один и тот же параметр. Например, правило, задающее цвет для всех тегов span, правило, задающее цвет для всех с классом blue, и какая-нибудь сложная конструкция с вложениями и соседством. Для каждого правила считается приоритет. Каждый тег имеет вклад 1, класс — 10, а идентификатор — 100. Для правила считается сумма вкладов его членов и используется правило с наибольшим вкладом. Если же таких несколько, то последнее из описанных в файле.

Чтобы понять, почему у вашего тега именно такой стиль, а не тот, который вы ожидали увидеть, удобно пользоваться консолью разработчика в браузере. Например, в Google Chrome она открывается путем нажатия на F12. Там же можно увидеть ошибки, которые произошли при попытке загрузить и применить ваш CSS файл.

Псевдоклассы

Множество тегов с выполненным определенным свойством. Записываются через двоеточие. Например, a:visited { color: olive; } позволяет выделить ссылки, по которым уже переходили, не фиолетовым цветом, а оливковым. Чтобы сделать что-то только с непосещенными ссылками, следует использовать a:link.

С помощью псевдоклассов можно задать определенный стиль тегу, когда на него наведен курсор мыши. Для этого используется класс :hover. Например, пока что так изменяется цвет у заголовков на этой странице: h2:hover { color: dimgrey; }

Box Model

Контент — содержимое тега, например, текст. Ширина и высота задаются именно для него. Из-за этого суммарная ширина тега может превысить 100%, с этим нужно быть аккуратно.

Padding — отступ между содержимым тега и его границей.

Граница — может быть разной ширины, цветов, типов. В таблицах, если хотим, чтобы у клеточек были границы, сделать их видимыми недостаточно. Тогда мы увидим двойную границу (у каждой из клеточек). Поэтому требуется задать border: collapse

Margin — отступ вокруг границы, отделяющий тег от окружающих. У двух элементов, соседних по вертикали, внешние отступы могут склеиваться и заменяться на максимум из их отступов.

У тегов есть свойство box-sizing, по умолчанию равное content-box. Можно изменить его на border-box и граница с padding'ом будут включаться в ширину и высоту.

margin: auto используется, чтобы центрировать элемент по горизонтали.

Если задать абсолютную ширину, то при уменьшении размера экрана появится полоса прокрутки. Избежать этого можно с помощью свойств max-width и min-width. Обратите внимание, что они перезаписывают width!

С помощью float можно сделать галерею картинок, которая будет сама адаптироваться к изменениям размера окна.

Отображение, позиционирование

То, что происходит с тегом, когда его содержимое начинает превышать его размер, задается параметром overflow. Разберем несколько примеров:

Если задано значение hidden, то не поместившееся содержимое будет убрано. Вот это самое не поместившееся содержимое
Если задано значение visible, то не поместившееся содержимое будет показано. Вот это самое не поместившееся содержимое
Если задано значение scroll, то будут добавлены полосы прокрутки по обоим измерениям. Сделаем текст немножко длиннее, чтобы увидеть их использование.
Если задано значение auto, то в случае надобности будут добавлены полосы прокрутки по соответствующим измерениям.
Наведи на меня
Спрятанный текст

Теги не обязаны всегда быть строчными или блочными. Переопределяя свойство display можно сделать блочные гиперссылки или строчные элементы списка, что позволяет делать горизонтальные меню. Задавая none можно спрятать тег со страницы, что будет полезно, когда мы займемся JS или для создания выпадающих меню.

С помощью уже известных инструментов можно сделать красивое меню, но как закрепить его наверху или сбоку? У тегов есть параметр position, он используется для управления положением тега на странице. По умолчанию его значением задано static, тег занимает место в соответствии с разметкой страницы.

  • static: тег занимает свое место на странице.
  • relative: рассчитывается место тега на странице и к нему добавляется сдвиг, задаваемый параметрами left, top, right, bottom.
  • fixed: положение тега не зависит от прокрутки и другого содержимого страницы, а задается параметрами left, top, right, bottom.
  • absolute: положение тега относительно тега, его включающего, который был не static.
